# Errors, Reconnect & Help

Status: **Sketch** | Phase: 2

This screen doc is conceptual. Canonical UX gates are the journey specs in `docs/02_architecture/tui/journeys/` plus [Canvas + X-ray](../07_canvas_and_xray.md).

## Purpose

Handle error states, connection issues, and provide user guidance. These overlays appear when something goes wrong or when users need help.

## Entry Conditions

- Connection lost to server
- Tool execution fails
- User presses `?` for help
- Rate limit or quota exceeded

## Capabilities Used

| Capability | Usage |
|------------|-------|
| `ui.shortcuts` | Help display |
| Connection management | Reconnect logic |
| `openresponses.errors` | Error display |

|-----|--------|---------|

| `w` | Wait and retry | Rate limit |
| `c` | Cancel | Any error |
| `q` | Quit | Connection error |
| `l` | View log | Tool errors |
| `y` | Copy error | Any error |
| `?` | Details | Expand error info |
| `Esc` | Dismiss | Notifications |
| `Enter` | Action/Dismiss | Context-dependent |

---

## Considerations for Implementers

- **Error categorization**: Map provider errors to user-friendly messages.
- **Reconnect strategy**: Exponential backoff with jitter.
- **State preservation**: Don't lose user input during reconnect.
- **Help searchability**: Allow searching within help content.
- **Toast management**: Queue and manage multiple notifications.
